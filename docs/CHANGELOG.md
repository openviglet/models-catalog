## Block B — Client libraries / SDKs

> Thin, read-only client libraries that wrap the published endpoint so consumers
> don't hand-roll HTTP against the JSON. Each mirrors the repo's zero/minimal-dependency
> ethos, consumes the public artifacts, and shares one surface
> (`load`/`refresh`/`all`/`byKind`/`byVendor`/`get` + in-memory cache with optional TTL).

- **T9** — **JavaScript / TypeScript client library.** `clients/js/` — a zero-dependency npm package (`@openviglet/model-catalog-client`, browser + Node ≥ 18 on the global `fetch`) exposing the shared surface plus faceted-slice loaders (`fetchByKind`/`fetchByVendor`) and the `endpoints()` manifest. Plain ESM + a hand-written `index.d.ts` (typed `ModelEntry`/`Kind`/`CatalogEnvelope`, unknown fields tolerated for additive-schema safety — no build step); flattens the `vendors` map (backfilling `vendor`), selectable rolling/`pinnedVersion`/`compact` source, injectable `fetch`/`now`. 10 `node:test` units (fully offline via a fake fetch + injected clock). Read-only, no pricing. → §VI
- **T10** — **Python client library.** `clients/python/` — a stdlib-only pip package (`openviglet-model-catalog-client`, `urllib` — no `requests`) with the same surface as the JS client (`load`/`refresh`/`all`/`by_kind`/`by_vendor`/`get`/`vendors` + `fetch_by_kind`/`fetch_by_vendor`/`endpoints`). Typed `ModelEntry` `dataclass` mapping camelCase JSON → snake_case attrs, unknown keys kept in `.extra` for additive-schema safety; flattens the `vendors` map (backfilling `vendor`), selectable rolling/`pinned_version`/`compact` source, in-memory cache with optional TTL, injectable `fetch`/`now`. 10 `unittest` units (fully offline via a fake fetch + injected clock). Python ≥ 3.8, read-only, no pricing. → §VI
- **T11** — **Java client library.** `clients/java/` — a Maven artifact (`io.github.openviglet:model-catalog-client`) with **no runtime dependencies**: the JDK `java.net.http.HttpClient` plus a tiny hand-rolled zero-dep JSON reader (`Json`), no HTTP/JSON framework. Typed `record`s (`ModelEntry`/`Modalities`/`Catalog`) + a lenient `Kind` enum (unknown → `UNKNOWN`), unknown JSON keys kept in `ModelEntry.extra()` for additive-schema safety. Same surface as the JS/Python clients — `load`/`refresh`/`all`/`byKind`(Kind|String)/`byVendor`/`get`(→`Optional`)/`vendors` + `fetchByKind`/`fetchByVendor`/`endpoints` — via a fluent `Builder` (rolling/`pinnedVersion`/`compact`, TTL, timeout, pluggable `Fetcher`, injectable clock); checked fetch failures wrapped as `CatalogException`. 11 JUnit 5 units (test-scoped, fully offline via a fake fetcher + injected clock). Java 17+, read-only, no pricing. → §VI
- **T12** — **Auth-free GitHub distribution for the clients.** Turned the "once published" install notes into working, unauthenticated paths off the public repo: root `jitpack.yml` builds the `clients/java/` module on demand and pins it to `$VERSION` so JitPack serves it as `com.github.openviglet.model-catalog:model-catalog-client:<tag>` (no root aggregator pom, no login); `.github/workflows/release-js-client.yml` packs `openviglet-model-catalog-client-<v>.tgz` on a `js-v*` tag and attaches it to the matching GitHub Release for `npm install <asset-url>`. Python already installs via `pip install "git+https://…#subdirectory=clients/python"`. All three READMEs document the per-ecosystem no-auth path + the GitHub-Packages-needs-a-token / no-pip-index caveats, and the browsable page gained a **Client SDKs** section. → §VI
- **T13** — **Manual publish workflows: client SDKs → their public registries.** One input-free `workflow_dispatch` publisher per ecosystem, Turing-style (`publish-java-client.yml` · `publish-js-client.yml` · `publish-python-client.yml`). Each run **auto-increments the patch** in the ecosystem manifest (`pom.xml`/`package.json`/`pyproject.toml`), commits + tags the bump (`[skip ci]`; `java-v*`/`js-v*`/`py-v*` for provenance + JitPack fallback), runs the client's own tests as a publish gate, then publishes to the registry: **npm** (`npm publish --access public`, `@openviglet/model-catalog-client`), **PyPI** (twine sdist+wheel, `openviglet-model-catalog-client`), and **Maven Central** via the Central Portal (`mvn -P release deploy`). The Java `release` profile adds signed sources+javadoc (`maven-source`/`maven-javadoc`/`maven-gpg` + `central-publishing-maven-plugin`), kept out of the default build so `mvn verify` needs no GPG; the pom gained `<developers>` + `developerConnection`. Version is derived not typed, so it can't drift and each registry rejects a re-publish. Token-secret auth, Turing-aligned names (`NPM_TOKEN`; `PYPI_API_TOKEN`; `OSSRH_USERNAME`/`OSSRH_TOKEN` + `GPG_PRIVATE_KEY`/`GPG_PASSPHRASE`, via a manual GPG import + `s4u/maven-settings-action` on the `central` server). Consolidated the tag-triggered `release-js-client.yml` (T12) into the manual sibling; READMEs now show present-tense registry installs; `jitpack.yml` retained as a Java fallback. → §VI

## Block C — Web experience & discoverability

> Turn the browsable `public/index.html` from a read-only table into a tool developers
> reach for and link to. Every item is inline, zero-dependency and static-hosting-safe
> (no build step, no runtime). Rationale lived in [IMPROVEMENTS.md](IMPROVEMENTS.md) §C.

- **T16** — **URL-addressable explorer state + model permalinks.** The page now serializes its search + kind filter into the URL hash (`#q=…&kind=…`, written with `replaceState` so keystrokes don't spam history) and gives every row a stable, citable permalink (`#<vendor>/<id>`) — a hover `#` affordance copies the absolute URL and, on load or navigation, scrolls to and briefly flashes the target row. A single `hashchange` router disambiguates the three hash shapes (state `…=…`, model `…/…`, bare section anchor) so back/forward and shared deep links restore the exact view or focus a single model. Handles ids with `:`/`/`; zero-dep, inline, no schema or API change. → §C1
- **T17** — **Model detail drawer.** Clicking a row (or opening a `#vendor/id` permalink) now slides in a right-side drawer rendering the *full* record — vendor, kind, context/output windows (exact + humanized), embedding dims, input/output modalities, capabilities, and the schema-optional `status`/`knowledgeCutoff`/`releaseDate`/`aliases` (rendered only when present) — with provenance (`sources` + `lastVerified`) foregrounded in a highlighted panel, plus **copy-id** and **copy-as-JSON** actions. Wired through the T16 hash router (drawer open *is* the model permalink); closes via ✕/Esc/backdrop, restoring the list URL so the row is re-openable; back/forward closes it. Read-only, purely presentational, zero-dep, no schema change. → §C2
- **T18** — **Model comparison view.** Pin 2–4 models (a per-row ⇄ toggle or the drawer's Compare action) into a bottom tray, then open a side-by-side modal with an attributes×models table — vendor, kind, context/output windows (humanized + exact), embedding dims, input/output modalities, capabilities, lifecycle (`status`/`knowledgeCutoff`/`releaseDate`), and provenance (`sources`/`lastVerified`); rows no pinned model has are hidden. The comparison is itself a shareable deep link (`#compare=vendor/id,vendor/id`) routed through the T16 hash router, with per-column ✕ removal (auto-closing below 2) and ✕/Esc/backdrop close that restores the list URL. Client-side over the loaded catalog; zero-dep, no schema change. → §C3
- **T19** — **Command palette (⌘K / Ctrl-K).** A keyboard-first fuzzy quick-switcher (also reachable from a nav ⌘K button) over every model, vendor and kind, built from a one-pass index of the loaded catalog. A subsequence scorer (contiguity + shorter-match bonus) ranks results; ↑/↓ navigate, ↵ opens, esc closes; an empty query lists the kinds + vendors as quick-jumps. A pick is a thin layer over existing state — a model navigates to its `#vendor/id` permalink (opening the T17 drawer), a kind applies the kind filter, a vendor scrolls to its section — so nothing new is persisted. Inline, zero-dep. → §C4
- **T20** — **Capability & modality filters + sortable columns.** Beyond the kind filter, the explorer now derives capability, input-modality and output-modality filter chips straight from the loaded catalog (so they never drift), applied with AND semantics; and the table's column headers are click/keyboard sortable within each vendor group — Model (id), Kind, and a Details header that cycles context-window↓/↑ then max-output↓/↑ then off, with `aria-sort` + arrow indicators. Both filters and the active sort serialize into the T16 hash (`cap`/`in`/`out`/`sort`), so a filtered-and-sorted view is a shareable deep link. Pure client-side over the loaded catalog; zero-dep, no API change. → §C5
- **T21** — **Insights dashboard.** A new hero-adjacent "At a glance" section renders six zero-dependency **inline-SVG** horizontal bar charts — models per vendor, models per kind (kind-coloured), context-window distribution, top capabilities, input modalities, and per-field coverage (as %) — fed by the emitted `stats.json` (T24), except the context-window histogram which is bucketed client-side from the already-loaded catalog. Charts are theme-aware (bar fills via inline `style` so CSS `var()` colours resolve; labels/tracks via CSS classes) and the section stays hidden if `stats.json` is absent (old deploy), so it degrades cleanly. A nav "Insights" link jumps to it. Inline, zero-dep, no schema/API change. → §C6

## Block D — API surface expansion

> More ways to consume the same canonical file, all emitted deterministically at publish
> time — no server, no query runtime, no new dependency. Rationale lived in
> [IMPROVEMENTS.md](IMPROVEMENTS.md) §D.

- **T22** — **Catalog change feed.** `scripts/emit.mjs` now also emits `public/changes.json` + an Atom `public/feed.xml` — the delta between the freshly built catalog and the previously published one: `added` / `removed` / lifecycle-`changed` ids for this publish (each `{ vendor, id, kind, label }`; `changed` also carries `from`/`to` effective status, comparing `status` with a `deprecated → DEPRECATED` fallback), a `counts` rollup, and `previousLastUpdated` + a `baseline` flag. The baseline resolves prior-on-disk-build → live published `catalog.json` (best-effort `fetch`, 8s timeout, `CATALOG_EMIT_NO_FETCH` to force offline) → none (first publish → empty diff, `"baseline": "none"`). The Atom feed has one deterministic `<entry>` per change (timestamps derived from `lastUpdated`, no wall-clock read, so it's diff-friendly), each linking the model's site permalink. Advertised in `endpoints.json` (`changes`/`feed`), the page API table and `api.md`. Diff-at-emit so it can never drift; zero-dep, canonical file + schema untouched. → §D1
- **T23** — **Alternate export formats.** `scripts/emit.mjs` now also emits `public/catalog.csv` + `public/catalog.ndjson` from the same flattened entries as `catalog.json`, in the same order (so they can't drift). CSV is one row per model with a fixed, stable column set (`vendor,id,label,kind,contextWindow,maxOutputTokens,embeddingDimensions,capabilities,inputModalities,outputModalities,knowledgeCutoff,releaseDate,status,deprecated,aliases,sources,lastVerified`) — array fields `;`-joined, RFC-4180 quoting (only cells containing `",\r\n` are quoted, inner `"` doubled), `\r\n` line endings, `modalities` split into input/output columns. NDJSON is one flattened `ModelEntry` (including `vendor`) per `\n`-delimited line, for streaming / `jq -c` / `grep`. Advertised in `endpoints.json` (`csv`/`ndjson`), the page API table and `api.md` (with `awk`/`jq` examples). Confirmed the CSV column set that kept this an idea. Zero-dep, canonical file + schema untouched. → §D2
- **T25** — **Extended faceting + alias resolution.** `scripts/emit.mjs` now also emits `public/by-capability/<cap>.json` and `public/by-modality/<m>.json` — the catalog filtered to one capability (`reasoning`/`vision`/`tools`/`multimodal`) or one modality present on **input or output** (union, direction-agnostic — the exact split stays in `stats.json`) — each keeping the shared `vendors`-map envelope plus a narrowing `capability`/`modality` field (still schema-valid), mirroring the T8 by-kind/by-vendor slices. Present capabilities/modalities are derived from the flattened entries so the slice set never drifts, and the two dirs are rewritten each run so a vanished facet leaves no stale file. Also emits `public/aliases.json` — an `alias id → { vendor, id }` resolution map (sorted, with a `count`; collisions warn + keep-first) so a consumer resolving a `-latest`/dated snapshot needn't scan every entry; empty today (no entry carries `aliases` yet) and auto-populates at emit when they do. All advertised in `endpoints.json` (`byCapability`/`byModality`/`aliases`), the page API table and `api.md`. Zero-dep, canonical file + schema untouched. → §D4
- **T26** — **GEO / citability artifacts.** `scripts/emit.mjs` now also emits, from the same flattened entries, a citability layer so the catalog can be indexed and *cited* (STRATEGY §I): `public/llms.txt` (the [llms.txt](https://llmstxt.org) convention — title + summary + linked sections for the catalog data, every vendor and every model) and a real, indexable URL per vendor (`models/<vendor>/` index) and per model (`models/<vendor>/<slug>.md` + `.html`). Each model page carries a prose summary and a facts table (context/output windows humanized + exact, embedding dims, input/output modalities, capabilities, lifecycle, provenance — present fields only); the Markdown page is the quotable primary artifact, the minimal self-contained HTML page adds a `<title>`, meta description, canonical + OG tags and a theme-aware stylesheet. Model ids (which carry `/` and `:` for bedrock/ollama/hf) are slugged to safe, unique, lower-case file names, memoized per id so links and files always agree; the `models/` tree is rewritten each run so a vanished model leaves no stale page. Advertised in `endpoints.json` (`llms`/`pages`), the page API table and `api.md`. 194 model + 14 vendor pages + llms.txt; zero-dep, canonical file + schema untouched. → §D5
- **T27** — **Embeddable status badge.** `scripts/emit.mjs` now also emits `public/badge.json` — a [shields.io endpoint](https://shields.io/badges/endpoint-badge) payload (`{ schemaVersion: 1, label: "Model Catalog", message: "N models · M vendors", color: "ea580c" (Viglet orange), cacheSeconds: 3600 }`) so any README can render a live badge via `img.shields.io/endpoint?url=…/badge.json` that always reflects the current totals. Reads the same `totals` as `stats.json`, computed at emit, so it never goes stale. Advertised in `endpoints.json` (`badge`), the page API table and `api.md`; the browsable page gained an **Embed a live badge** card (live preview + copyable Markdown snippet). Zero-dep, canonical file + schema untouched. → §D6
- **T24** — **`stats.json` aggregate metrics.** `scripts/emit.mjs` now also emits `public/stats.json` — a small, separately-shaped artifact (its own envelope, not a `vendors` map) of pre-computed counts derived from the flattened catalog: grand `totals` (models / vendors / kinds / capabilities), the `byVendor` / `byKind` / `byCapability` / `byInputModality` / `byOutputModality` count maps (ranked by descending count then key, so the file is deterministic and diff-friendly), and per-field `coverage` — `{ filled, rate }` over all models for contextWindow / maxOutputTokens / embeddingDimensions / capabilities / modalities / knowledgeCutoff / releaseDate / aliases / status / sources / lastVerified. Advertised in `endpoints.json` (`stats`), the page's API table and `api.md`. Derived at emit so it can never drift from the catalog; unblocks the T21 dashboard (and later the T27 badge / T29 coverage view). Zero-dep, canonical file + schema untouched. → §D3

## Block F — Cost & commercial offerings

> Reverses the original identity-not-pricing bet (STRATEGY §I): the catalog now
> carries an **optional, indicative US list price** so **Viglet Turing ES** can
> produce consumption/spend reports without every consumer joining to a second
> source. Bounded — US list price only, optional, provenance-gated, never invented,
> flagged **indicative, not authoritative** (verify with the vendor). Rationale
> lived in [IMPROVEMENTS.md](IMPROVEMENTS.md) §F.

- **T30** — **Pricing field + schema (strategy reversal).** Added an optional `pricing` object to `ModelEntry` in `catalog/model-catalog.schema.json` — `inputPer1M`/`outputPer1M` (USD per 1M tokens, `minimum: 0`), `currency` (fixed `enum: ["USD"]`), `unit`, an explicit `indicative` (`const: true`), a free-text `note` disclaimer, and provenance `source` + `lastVerified` — with `required: ["indicative", "source", "lastVerified"]` so a price is admitted only when it is provenance-gated and self-flagged non-authoritative. Additive; `version` stays `1`.
