## Block A ‚Äî Catalog API + regeneration pipeline

- **T1** ‚Äî **Canonical catalog + JSON Schema + public API.** `catalog/models-catalog.json` versioned envelope (`version`/`lastUpdated`/`vendors`) with `kind` on every entry + optional `contextWindow`/`embeddingDimensions`/`capabilities`/`deprecated`; Draft-2020-12 `catalog/models-catalog.schema.json`; zero-dep `scripts/emit.mjs` validates + emits the public artifacts (`catalog.json` + pinned `catalog-v1.json` + schema, each entry flattened with its `vendor`) ‚Äî 11 vendors, 63 models. - **T2** ‚Äî **Multi-source regeneration pipeline.** `pipeline/` zero-dep Node pipeline that regenerates the canonical file from live vendor APIs (OpenAI/Anthropic/Gemini/Cohere/Mistral) + the LiteLLM registry (non-pricing enrichment) + a curated `overrides.json`: `SourceAdapter` contract + offline snapshot cache, merge/reconciliation (precedence pin>live>override>committed>litellm, per-field provenance + conflicts, carry-forward + positive-evidence removal + anchoring), validation + diff review gate (`--apply` only), `npm run regen` orchestrator, and 10 `node:test` units. Additive schema fields `maxOutputTokens`/`modalities`/`knowledgeCutoff`/`releaseDate`/`aliases`/`status` + provenance `sources`/`lastVerified`. (orig. Turing BD / T764‚ÄìT770) ‚Üí ¬ß‚Äî
- **T5** ‚Äî **Opt-in CI regeneration ‚Üí PR.** `.github/workflows/regen.yml` ‚Äî a manual-dispatch (`workflow_dispatch`) workflow that runs the pipeline with API keys from repo secrets, applies the result on a throwaway `catalog-regen/<run>` branch, and opens a PR with the diff report as the body. Never writes to `main`, never auto-merges ‚Äî a human reviews adds/removes/conflicts and merges. The safe way to let CI refresh the catalog. ‚Üí ¬ß‚Äî
- **T4** ‚Äî **Extended self-hosted / aggregator sources.** Three new opt-in `SourceAdapter`s so non-cloud vendor rows stop depending on LiteLLM alone for anchoring: `ollama-api` (`GET /api/tags` on `$OLLAMA_HOST`), `bedrock-api` (`ListFoundationModels`, AWS SigV4 hand-rolled on `node:crypto` ‚Äî zero-dep, no aws-sdk; modalities‚Üíkind, `modelName` label, lifecycle‚Üístatus), and `huggingface-api` (Hub sentence-transformers ‚Üí local ONNX embedding models). Each is gated on its own env var (missing ‚Üí skipped, never fails) and flagged **`partial`**: it *anchors* the ids it returns but ‚Äî being environment-scoped/bounded ‚Äî is never positive-evidence for removal (new `AGGREGATOR_PRIORITY` 20, below curated overrides). 3 new normalize unit tests (13 total). ‚Üí ¬ß‚Äî
- **T8** ‚Äî **Faceted static slices + discovery manifest.** `scripts/emit.mjs` now also writes `public/by-kind/<KIND>.json` and `public/by-vendor/<vendor>.json` ‚Äî the catalog filtered to one kind/vendor (same `vendors`-map envelope + a narrowing `kind`/`vendor` field; still schema-valid) ‚Äî plus `endpoints.json`, a machine-readable manifest of every published path as absolute URLs (`latest`/`pinned`/`index`/`schema`/`byKind`/`byVendor`). Slice dirs are rewritten each run so a removed kind/vendor leaves no stale file. Documented in `api.md`; the static-first answer to "more query methods" (a dynamic query API stays out of scope for Pages). ‚Üí ¬ß‚Äî
- **T7** ‚Äî **Compact index endpoint.** `scripts/emit.mjs` now also writes `public/index.json` ‚Äî the same envelope shape but each entry trimmed to `{ vendor, id, label, kind }` (all dropped fields are optional, so it still validates against the catalog schema). ~72% smaller than `catalog.json`, for model-pickers that only render a grouped list and lazy-load the full record on selection. Documented in `api.md` + the browsable page; zero-dep, canonical file + envelope schema untouched. ‚Üí ¬ß‚Äî
- **T3** ‚Äî **Repo extraction + publishing + project docs.** Stood up this standalone `openviglet/models-catalog` repo: relocated the canonical data + schema + pipeline + emit; a self-contained browsable `public/index.html`; a GitHub Actions **Publish Pages** workflow (emit ‚Üí deploy, never regenerates upstream); the reference docs (`docs/references/api.md`, `pipeline.md`); the Turing-style roadmap set (ROADMAP/CHANGELOG/IMPROVEMENTS/STRATEGY/last-task) + `roadmap-docs` skill; README + agents.md + CLAUDE.md + Apache-2.0. - **T14** ‚Äî **New frontier vendor rows: DeepSeek, xAI, MiniMax, Z.ai (GLM).** Mapped the four native `litellm_provider`s (`deepseek`/`xai`/`minimax`/`zai`) in the LiteLLM adapter so their registry entries stop being dropped, and seeded curated ids in `overrides.json` (the anchoring source LiteLLM can enrich): `deepseek-chat`/`deepseek-reasoner`, `grok-4`/`grok-3`/`grok-3-mini`/`grok-code-fast-1`, `MiniMax-M2`/`MiniMax-M3`, `glm-4.6`/`glm-4.5`/`glm-4.5-air` ‚Äî 11 models across 4 new vendors, all CHAT, contextWindow/maxOutputTokens/capabilities/modalities filled by LiteLLM. ## Block B ‚Äî Client libraries / SDKs

> Thin, read-only client libraries that wrap the published endpoint so consumers
> don't hand-roll HTTP against the JSON. Each mirrors the repo's zero/minimal-dependency
> ethos, consumes the public artifacts, and shares one surface
> (`load`/`refresh`/`all`/`byKind`/`byVendor`/`get` + in-memory cache with optional TTL).

- **T9** ‚Äî **JavaScript / TypeScript client library.** `clients/js/` ‚Äî a zero-dependency npm package (`@openviglet/models-catalog-client`, browser + Node ‚â• 18 on the global `fetch`) exposing the shared surface plus faceted-slice loaders (`fetchByKind`/`fetchByVendor`) and the `endpoints()` manifest. Plain ESM + a hand-written `index.d.ts` (typed `ModelEntry`/`Kind`/`CatalogEnvelope`, unknown fields tolerated for additive-schema safety ‚Äî no build step); flattens the `vendors` map (backfilling `vendor`), selectable rolling/`pinnedVersion`/`compact` source, injectable `fetch`/`now`. 10 `node:test` units (fully offline via a fake fetch + injected clock). Read-only, no pricing. ‚Üí ¬ßVI
- **T10** ‚Äî **Python client library.** `clients/python/` ‚Äî a stdlib-only pip package (`openviglet-models-catalog-client`, `urllib` ‚Äî no `requests`) with the same surface as the JS client (`load`/`refresh`/`all`/`by_kind`/`by_vendor`/`get`/`vendors` + `fetch_by_kind`/`fetch_by_vendor`/`endpoints`). Typed `ModelEntry` `dataclass` mapping camelCase JSON ‚Üí snake_case attrs, unknown keys kept in `.extra` for additive-schema safety; flattens the `vendors` map (backfilling `vendor`), selectable rolling/`pinned_version`/`compact` source, in-memory cache with optional TTL, injectable `fetch`/`now`. 10 `unittest` units (fully offline via a fake fetch + injected clock). Python ‚â• 3.8, read-only, no pricing. ‚Üí ¬ßVI
- **T11** ‚Äî **Java client library.** `clients/java/` ‚Äî a Maven artifact (`io.github.openviglet:models-catalog-client`) with **no runtime dependencies**: the JDK `java.net.http.HttpClient` plus a tiny hand-rolled zero-dep JSON reader (`Json`), no HTTP/JSON framework. Typed `record`s (`ModelEntry`/`Modalities`/`Catalog`) + a lenient `Kind` enum (unknown ‚Üí `UNKNOWN`), unknown JSON keys kept in `ModelEntry.extra()` for additive-schema safety. Same surface as the JS/Python clients ‚Äî `load`/`refresh`/`all`/`byKind`(Kind|String)/`byVendor`/`get`(‚Üí`Optional`)/`vendors` + `fetchByKind`/`fetchByVendor`/`endpoints` ‚Äî via a fluent `Builder` (rolling/`pinnedVersion`/`compact`, TTL, timeout, pluggable `Fetcher`, injectable clock); checked fetch failures wrapped as `CatalogException`. 11 JUnit 5 units (test-scoped, fully offline via a fake fetcher + injected clock). Java 17+, read-only, no pricing. ‚Üí ¬ßVI
- **T12** ‚Äî **Auth-free GitHub distribution for the clients.** Turned the "once published" install notes into working, unauthenticated paths off the public repo: root `jitpack.yml` builds the `clients/java/` module on demand and pins it to `$VERSION` so JitPack serves it as `com.github.openviglet.models-catalog:models-catalog-client:<tag>` (no root aggregator pom, no login); `.github/workflows/release-js-client.yml` packs `openviglet-models-catalog-client-<v>.tgz` on a `js-v*` tag and attaches it to the matching GitHub Release for `npm install <asset-url>`. Python already installs via `pip install "git+https://‚Ä¶#subdirectory=clients/python"`. All three READMEs document the per-ecosystem no-auth path + the GitHub-Packages-needs-a-token / no-pip-index caveats, and the browsable page gained a **Client SDKs** section. ‚Üí ¬ßVI
- **T13** ‚Äî **Manual publish workflows: client SDKs ‚Üí their public registries.** One input-free `workflow_dispatch` publisher per ecosystem, Turing-style (`publish-java-client.yml` ¬∑ `publish-js-client.yml` ¬∑ `publish-python-client.yml`). Each run **auto-increments the patch** in the ecosystem manifest (`pom.xml`/`package.json`/`pyproject.toml`), commits + tags the bump (`[skip ci]`; `java-v*`/`js-v*`/`py-v*` for provenance + JitPack fallback), runs the client's own tests as a publish gate, then publishes to the registry: **npm** (`npm publish --access public`, `@openviglet/models-catalog-client`), **PyPI** (twine sdist+wheel, `openviglet-models-catalog-client`), and **Maven Central** via the Central Portal (`mvn -P release deploy`). The Java `release` profile adds signed sources+javadoc (`maven-source`/`maven-javadoc`/`maven-gpg` + `central-publishing-maven-plugin`), kept out of the default build so `mvn verify` needs no GPG; the pom gained `<developers>` + `developerConnection`. Version is derived not typed, so it can't drift and each registry rejects a re-publish. Token-secret auth, Turing-aligned names (`NPM_TOKEN`; `PYPI_API_TOKEN`; `OSSRH_USERNAME`/`OSSRH_TOKEN` + `GPG_PRIVATE_KEY`/`GPG_PASSPHRASE`, via a manual GPG import + `s4u/maven-settings-action` on the `central` server). Consolidated the tag-triggered `release-js-client.yml` (T12) into the manual sibling; READMEs now show present-tense registry installs; `jitpack.yml` retained as a Java fallback. ‚Üí ¬ßVI
