## Block B вҖ” Client libraries / SDKs

> Thin, read-only client libraries that wrap the published endpoint so consumers
> don't hand-roll HTTP against the JSON. Each mirrors the repo's zero/minimal-dependency
> ethos, consumes the public artifacts, and shares one surface
> (`load`/`refresh`/`all`/`byKind`/`byVendor`/`get` + in-memory cache with optional TTL).

- **T9** вҖ” **JavaScript / TypeScript client library.** `clients/js/` вҖ” a zero-dependency npm package (`@openviglet/model-catalog-client`, browser + Node вүҘ 18 on the global `fetch`) exposing the shared surface plus faceted-slice loaders (`fetchByKind`/`fetchByVendor`) and the `endpoints()` manifest. Plain ESM + a hand-written `index.d.ts` (typed `ModelEntry`/`Kind`/`CatalogEnvelope`, unknown fields tolerated for additive-schema safety вҖ” no build step); flattens the `vendors` map (backfilling `vendor`), selectable rolling/`pinnedVersion`/`compact` source, injectable `fetch`/`now`. 10 `node:test` units (fully offline via a fake fetch + injected clock). Read-only, no pricing. вҶ’ В§VI
- **T10** вҖ” **Python client library.** `clients/python/` вҖ” a stdlib-only pip package (`openviglet-model-catalog-client`, `urllib` вҖ” no `requests`) with the same surface as the JS client (`load`/`refresh`/`all`/`by_kind`/`by_vendor`/`get`/`vendors` + `fetch_by_kind`/`fetch_by_vendor`/`endpoints`). Typed `ModelEntry` `dataclass` mapping camelCase JSON вҶ’ snake_case attrs, unknown keys kept in `.extra` for additive-schema safety; flattens the `vendors` map (backfilling `vendor`), selectable rolling/`pinned_version`/`compact` source, in-memory cache with optional TTL, injectable `fetch`/`now`. 10 `unittest` units (fully offline via a fake fetch + injected clock). Python вүҘ 3.8, read-only, no pricing. вҶ’ В§VI
- **T11** вҖ” **Java client library.** `clients/java/` вҖ” a Maven artifact (`io.github.openviglet:model-catalog-client`) with **no runtime dependencies**: the JDK `java.net.http.HttpClient` plus a tiny hand-rolled zero-dep JSON reader (`Json`), no HTTP/JSON framework. Same surface as the JS/Python clients вҖ” `load`/`refresh`/`all`/`byKind`(Kind|String)/`byVendor`/`get`(вҶ’`Optional`)/`vendors` + `fetchByKind`/`fetchByVendor`/`endpoints` вҖ” via a fluent `Builder` (rolling/`pinnedVersion`/`compact`, TTL, timeout, pluggable `Fetcher`, injectable clock); checked fetch failures wrapped as `CatalogException`. 11 JUnit 5 units (test-scoped, fully offline via a fake fetcher + injected clock). > Turn the browsable `public/index.html` from a read-only table into a tool developers
> reach for and link to. Every item is inline, zero-dependency and static-hosting-safe
> (no build step, no runtime). Rationale lived in [IMPROVEMENTS.md](IMPROVEMENTS.md) В§C.

- **T16** вҖ” **URL-addressable explorer state + model permalinks.** The page now serializes its search + kind filter into the URL hash (`#q=вҖҰ&kind=вҖҰ`, written with `replaceState` so keystrokes don't spam history) and gives every row a stable, citable permalink (`#<vendor>/<id>`) вҖ” a hover `#` affordance copies the absolute URL and, on load or navigation, scrolls to and briefly flashes the target row. A single `hashchange` router disambiguates the three hash shapes (state `вҖҰ=вҖҰ`, model `вҖҰ/вҖҰ`, bare section anchor) so back/forward and shared deep links restore the exact view or focus a single model. Handles ids with `:`/`/`; zero-dep, inline, no schema or API change. вҶ’ В§C1
- **T17** вҖ” **Model detail drawer.** Clicking a row (or opening a `#vendor/id` permalink) now slides in a right-side drawer rendering the *full* record вҖ” vendor, kind, context/output windows (exact + humanized), embedding dims, input/output modalities, capabilities, and the schema-optional `status`/`knowledgeCutoff`/`releaseDate`/`aliases` (rendered only when present) вҖ” with provenance (`sources` + `lastVerified`) foregrounded in a highlighted panel, plus **copy-id** and **copy-as-JSON** actions. Wired through the T16 hash router (drawer open *is* the model permalink); closes via вң•/Esc/backdrop, restoring the list URL so the row is re-openable; back/forward closes it. Read-only, purely presentational, zero-dep, no schema change. вҶ’ В§C2
- **T18** вҖ” **Model comparison view.** Pin 2вҖ“4 models (a per-row вҮ„ toggle or the drawer's Compare action) into a bottom tray, then open a side-by-side modal with an attributesГ—models table вҖ” vendor, kind, context/output windows (humanized + exact), embedding dims, input/output modalities, capabilities, lifecycle (`status`/`knowledgeCutoff`/`releaseDate`), and provenance (`sources`/`lastVerified`); rows no pinned model has are hidden. - **T19** вҖ” **Command palette (вҢҳK / Ctrl-K).** A keyboard-first fuzzy quick-switcher (also reachable from a nav вҢҳK button) over every model, vendor and kind, built from a one-pass index of the loaded catalog. A subsequence scorer (contiguity + shorter-match bonus) ranks results; вҶ‘/вҶ“ navigate, вҶө opens, esc closes; an empty query lists the kinds + vendors as quick-jumps. A pick is a thin layer over existing state вҖ” a model navigates to its `#vendor/id` permalink (opening the T17 drawer), a kind applies the kind filter, a vendor scrolls to its section вҖ” so nothing new is persisted. Inline, zero-dep. вҶ’ В§C4
