Model Catalog

A vendor-neutral, kind-aware catalog of LLM / embedding / rerank / media models — which ids exist per vendor and, for each, its kind, context window, and capabilities. Free, unauthenticated, versioned JSON. Part of Viglet Turing ES.

Public REST API

Static, CORS-open, versioned JSON — no key, no server. Every path below is a plain GET.

EndpointWhat you get
catalog.json Full catalog — every vendor, every field. Rolling latest.
catalog-v1.json Same, pinned to schema v1 — safe for consumers to lock onto.
index.json Compact index — each model trimmed to { vendor, id, label, kind }. ~70% smaller; ideal for model-pickers that lazy-load the full record on selection.
by-kind/<KIND>.json Catalog filtered to one kind — e.g. by-kind/EMBEDDING.json, CHAT.
by-vendor/<vendor>.json Catalog filtered to one vendor — e.g. by-vendor/openai.json.
endpoints.json Discovery manifest — absolute URLs for every path above, including the available byKind / byVendor slice keys. Read this instead of hard-coding paths.
catalog.schema.json JSON Schema (Draft 2020-12) describing the envelope + entry.

Faceted slices are pre-computed at publish time, not a live query API — GitHub Pages serves static assets only. Every response (full, index, or slice) shares the same vendors-map envelope, so one parser reads them all. 

# every embedding model, straight from the pre-filtered slice
curl -s ./by-kind/EMBEDDING.json | jq '.vendors | to_entries[].value[].id'

# discover the whole surface
curl -s ./endpoints.json | jq

Client SDKs

Zero-dependency, read-only clients that wrap the endpoints above — URL selection (rolling / pinned / compact), vendors-map flattening, byKind/byVendor/get filters and in-memory caching. All open, Apache-2.0, published to their public registries — every install below is unauthenticated, no token, anyone can.
